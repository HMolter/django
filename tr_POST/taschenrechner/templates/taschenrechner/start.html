Hier ist der Taschenrechner über eine einzige HTML Seite mit der POST Methode realisiert. Die Taste übermittelt vorher in einem Formular erstellte Daten. Hier könnte es z.B. noch Häkchen oder Texteingaben geben. Im Gegensatz zur GET Methode wird hier über eine Signatur (Ihr erinnert euch hoffentlich :-)), das CSRF Token, sichergestellt, dass niemand die Eingabe manipuliert oder abhört. Der Link oben im Formular führt erneut zur selben HTML Vorlage.

In der views.py Datei gibt es eine einzige Funktion, die mit Hilfe der übermittelten Daten die Vorlage umbaut und schließlich als HTML zurück gibt.
